Disable user stats? Gebruiker statistieken uitzetten?
User stats are maintained to help power shortcodes such as [weight-loss-tracker-league-table]. This involves a cron job being run every hour to refresh a user's stats such as first weight, recent weight and total lost. The figures are also re-produced on each weight entry modification and can then be used to generate a league table. This feature could <em>potentially</em> reduce performance on your site. If you are not using the leaderboard shortcodes it is advised to disable this. Gebruikers statistieken worden gebruikt om te helpen bij het gebruik van power shorcodes zoals [weight-loss-tracker-league-table]. Dit omvat een cron run die elk uur draait om gebruikers data zoals eerste gewicht, recent gewicht en totaal verloren te rerfeshen. De cijfers zijn ook opnieuw geproduceerd op elke gewichtsklasse invoer en aanpassingen kunnen vervolgens worden gebruikt om een ​ranglijst genereren. Deze functie kan <em> mogelijk </ em> prestaties van uw site verminderen. Als u geen gebruik maakt van het leaderboard shortcodes wordt geadviseerd om dit uit te schakelen.

By consenting, you are allowing permission for the following data to be sent on a weekly basis to YeKen: URL, summary of settings, count of recorded weights and measurements. No user or admin data will ever be transmitted. We houden ervan om te zien hoe onze plugin wordt gebruikt. Met het geven van goedkeuring, geef je toestemming om de volgende data op maandelijkse basis gestuurd naar YeKen: URL, samenvatting van instellingen, aantal ingevoerde gewichten en metingen. Er zal nooit gebruikers of admin data worden gestuurd.
